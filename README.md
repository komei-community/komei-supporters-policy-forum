# 公明サポーターズ政策協議リポジトリ

![GitHub](https://img.shields.io/badge/Status-試運転中-yellow)

## 📋 リポジトリの概要

このリポジトリは、公明党の政策形成プロセスにオープンな議論と透明性をもたらすためのプラットフォームです。GitHub を活用し、誰もが政策提案・議論・レビューに参加できる仕組みを提供します。

### 🎯 目的

- **政策提案のオープン化**: 従来の閉じられた政策形成を誰でも参加可能な透明なプロセスに
- **市民の声の集約**: 支援者・有権者の意見を政策に直接反映する仕組みの構築
- **実現性の高い提案**: 政策実現能力の高い公明党への直結した政策提案ルートの醸成

### 🚀 特徴

- **GitHub Issue/PR**: 政策提案・議論・レビューのワークフロー
- **透明性**: 全ての議論過程が公開・記録される
- **協働**: 複数の参加者による建設的な議論と改善
- **成果物公開**: GitHub Pages での採択済み政策の公開

## 📁 フォルダ構成

## 🤝 参加方法

### 💡 政策を提案したい場合

1. **Issue 作成**: 新規政策提案用の Issue テンプレートを使用
2. **議論参加**: 提案された Issue でコメント・改善案を投稿
3. **PR 作成**: 具体的な政策案を Pull Request で提出

### 📝 議論に参加したい場合

- 既存の Issue や Pull Request にコメント
- 建設的な意見・改善案・データ提供

### 🔍 詳細情報・関連リンク

- [プロジェクト憲章](./プロジェクト憲章.md): プロジェクトの理念・目的・スコープ
- [運用ルール](./運用ルール.md): 参加ルール・議論の進め方・成果物管理
- [編集ガイド](./HowToEdit.md): 具体的な参加・編集方法
- 公開用ポータルサイト（GitHub Pages）: このリポジトリの内容をGitHubアカウントがなくても閲覧できるWebサイト（URLは設定後にここに記載）

### 🌐 GitHub Pages による公開

- `web/` ディレクトリ以下に、GitHub Pages 向けの静的サイト（Jekyll）が配置されています。
- リポジトリの `main` ブランチに変更が push されると、GitHub Actions が自動で次を行います：
  - GitHub API から最新の Issue 一覧を取得し、要約付きのデータとして `web/_data/issues.json` を生成
  - `web/` をソースに Jekyll で静的サイトをビルド
  - ビルド結果を GitHub Pages 環境へデプロイ
- これにより、採択済み政策やIssue一覧を含むポータルサイトが自動更新されます。
