# プロジェクト憲章

## 1. 基本情報

- **プロジェクト名**: 公明サポーターズ政策協議リポジトリ
- **作成日**: 2025/09/20
- **作成者**: 掛川　浩一

## 2. 背景・目的

### 背景

従来の政策形成は閉じられた場で行われることが多く、市民や支援者が直接意見を反映する仕組みは限定的であった。GitHub を活用することで、誰もが政策形成のプロセスにアクセスし、透明性・参加性を高めることが可能となる。

### 目的

- 政策提案にかかる議論のオープン化
- 支援者・有権者の声を集約し、政策へ反映
- 政策実現能力の高い公明党に対する直結した政策提案ルートの醸成

## 3. スコープ

### 対象（In-Scope）

- 政策アイデアの提案、議論、レビュー
- GitHub を用いた提案管理（Issue/PR/ラベル運用）
- 成果物の公開（GitHub Pages、PDF/HTML レポート）

### 非対象（Out-of-Scope）

- 個別候補者の選挙戦略
- 法的拘束力を持つ最終決定権

## 4. 成果物（Deliverables）

- 政策提案ドラフト（`policy-drafts/`）
- 採択済み政策文書（`policy-docs/` → GitHub Pages 公開）
- 定期レポート（PDF/HTML：提案一覧・採択結果・レビューまとめ）
- 外部資料整理（新聞記事、マニフェストの Markdown 化）
- 研究資料（統計データ・参考文献）

## 5. 成功基準

- 提案プロセス（Issue→PR→Merge）が円滑に回ること
- 提案のうち一定数（例：四半期で 5 件以上）が「採択済み」として公開されること
- 公明党議員や関係者との定期共有・意見交換が継続されること
- コミュニティの参加者数・貢献数が増加すること
- 成果物が社会的に認知され、外部メディアや広報誌で取り上げられること

## 6. 主要ステークホルダー

- **スポンサー／承認者**: 公明党関係者・コミュニティ代表
- **プロジェクトマネージャー/承認者**: 運営責任者（GitHub リポジトリ管理）
- **コントリビューター**: 提案・コメント・調査協力

## 7. リソース・予算

- **技術基盤**: GitHub（リポジトリ、Pages、Actions）
- **人的リソース**: ボランタリー参加者、モデレーター、メンテナー
- **予算**: 当面はゼロベース運営（必要に応じてクラウドサービス利用料など）

## 8. スケジュール概要

- **プロジェクト立ち上げ**: 2025/09/20
- **試運転開始**: 2025/09/20
- **正式運用開始**: 2025/10/20
- **議員・関係者との初回オンライン意見交換**: 未定

## 9. リスク・制約・前提

- **リスク**: 荒らしや誹謗中傷の発生、政治的中立性への疑念
- **制約**: 公明党の理念との整合性、GitHub の利用規約
