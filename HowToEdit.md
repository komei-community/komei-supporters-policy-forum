# 参加・編集ガイド

このガイドでは、公明サポーターズ政策協議リポジトリに参加し、政策提案や議論に貢献する具体的な方法を説明します。

## 🚀 はじめに

このプロジェクトでは、GitHub を活用して政策提案・議論・レビューを行います。GitHub 初心者の方でも参加できるよう、ステップバイステップで説明します。

## 📋 参加前の準備

### 必要なもの

- GitHub アカウント（無料で作成可能）
- ウェブブラウザ（Chrome、Firefox、Safari 等）

### 事前に読んでおくべき文書

- [プロジェクト憲章](./プロジェクト憲章.md)：プロジェクトの理念と目的
- [運用ルール](./運用ルール.md)：参加ルールと議論の進め方
- [README.md](./README.md)：プロジェクト全体の概要

## 🎯 参加方法

### 1. 政策を提案したい場合

#### Step 1: Issue の作成

1. [Issues タブ](../../issues)をクリック
2. 「New issue」ボタンをクリック
3. 政策提案用のテンプレートを選択
4. 以下の項目を記入：
   - **タイトル**: 提案の概要（例：「中小企業デジタル化支援強化策」）
   - **背景・課題**: なぜこの政策が必要か
   - **提案内容**: 具体的な政策案
   - **期待効果**: 実現により期待される成果
   - **参考資料**: 関連する資料やデータのリンク

#### Step 2: ブランチの自動作成

- Issue を作成すると、自動的に作業用ブランチ（`issue/番号-タイトル`）が作成されます
- このブランチで政策ドキュメントを作成します

#### Step 3: 政策ドキュメントの作成

1. 作成されたブランチに移動
2. 適切なフォルダに政策ファイルを作成：
   - **思索段階**: `draft-policy/` フォルダ
   - **本提案**: `policy/` フォルダ
3. ファイル名の形式：`YYYY-MM-DD_政策名.md`
4. 以下のテンプレートを使用：

```markdown
# [政策タイトル]

## 1. 背景・課題

[現状の問題点や課題を記述]

## 2. 提案内容

[具体的な政策案を記述]

## 3. 実施方法

[どのように実現するかの具体的手順]

## 4. 期待効果

[実現により期待される成果]

## 5. 実現可能性

[予算、法制度面での実現可能性]

## 6. 参考資料

[関連資料、データ、研究結果等]
```

#### Step 4: Pull Request の作成

1. 政策ドキュメントが完成したら「Pull Request」を作成
2. PR テンプレートに従って情報を記入：
   - **背景**: なぜこの提案が必要か
   - **提案内容**: 政策の概要
   - **期待効果**: 実現による効果
3. レビュアーを指定（運営管理者が設定）

### 2. 既存の提案に議論で参加したい場合

#### Issue での議論参加

1. [Issues タブ](../../issues)から興味のある提案を選択
2. コメント欄に建設的な意見を投稿：
   - **改善案**: より良くするための提案
   - **データ提供**: 関連する統計や事例
   - **実現性検討**: 実現可能性に関する意見
   - **参考情報**: 関連資料の共有

#### Pull Request でのレビュー参加

1. [Pull requests タブ](../../pulls)から提案を選択
2. 「Files changed」タブで具体的なコメント
3. レビュー観点：
   - **実現可能性**: 予算・法制度面での実現性
   - **新規性・独自性**: 既存政策との差別化
   - **期待効果**: 効果の妥当性と測定可能性

### 3. 外部資料の整理に貢献したい場合

#### 資料の Markdown 化

1. `sources/` フォルダに外部資料を整理
2. 新聞記事、マニフェスト、統計データ等を Markdown 形式に変換
3. ファイル名：`YYYY-MM-DD_情報源_タイトル.md`

#### フォルダ構成例

```
sources/
├── news/           # 新聞記事
├── manifesto/      # マニフェスト
├── statistics/     # 統計データ
└── research/       # 研究資料
```

## 🏷️ ラベルの理解

提案には以下のラベルが付与されます：

- **提案**: 新規提案
- **レビュー中**: 検討・議論中
- **採択済**: 採択され`adopted-policy/`に移動
- **見送り**: 今回は見送り（履歴は保持）
- **要調整**: 修正が必要

## 📝 コミュニケーションのガイドライン

### 建設的な議論のために

- **批判には代替案を**: 問題点の指摘とともに改善案を提示
- **データに基づく議論**: 感情論ではなく事実・データを重視
- **相手を尊重**: 人格攻撃や差別的発言は禁止

### 良いコメントの例

```
この提案の実現可能性について、以下の点で課題があると思います：

1. 予算面：年間○○億円の財源確保が必要
2. 法制度：○○法の改正が前提となる

代替案として、段階的実施（第1段階：パイロット事業、第2段階：全国展開）を提案します。
参考資料：[関連する既存施策のURL]
```

## 🔄 提案から採択までのフロー

```
1. Issue作成（政策アイデア提案）
   ↓
2. 自動でブランチ作成
   ↓
3. 政策ドキュメント作成
   ↓
4. Pull Request作成(2,3,4はIssueをGitHub Copilotにアサインすることで自動化可能)
   ↓
5. コミュニティレビュー・議論
   ↓
6. 運営管理者による最終レビュー
   ↓
7. マージ・採択 または 見送り
   ↓
8. 採択の場合：adopted-policy/に保存
   ↓
9. GitHub Pagesで公開（自動）
```

## 🔍 よくある質問

### Q: GitHub 初心者でも参加できますか？

A: はい。このガイドに従えば、ウェブブラウザだけで参加可能です。

### Q: どんな政策でも提案できますか？

A: 公明党の理念に沿った政策提案が対象です。詳細は[プロジェクト憲章](./プロジェクト憲章.md)をご確認ください。

### Q: 匿名で参加できますか？

A: GitHub アカウントが必要ですが、実名である必要はありません。ただし、建設的な議論ができるアカウントでの参加をお願いします。

### Q: 提案が採択される基準は？

A: 実現可能性、新規性・独自性、期待効果を総合的に判断します。詳細は[運用ルール](./運用ルール.md)をご参照ください。

---

**はじめての方へ**: まずは既存の Issue や Pull Request を読んで、議論の雰囲気を把握することをお勧めします。その後、小さなコメントから参加を始めてみてください。

**継続参加をお考えの方へ**: 定期的な議論参加、質の高い提案、建設的なレビューを通じて、コミュニティの発展にご協力ください。

## 🌐 GitHub Pages ポータルの仕組み

- このリポジトリには、GitHub アカウントがない方でも内容を閲覧できるようにするための公開用ポータルサイトが用意されています。
- `web/` ディレクトリに Jekyll ベースのサイトがあり、次の内容が公開されます：
  - README の概要
  - 提案中・採択済み政策へのリンク
  - Issue の一覧（タイトル・状態・作成日・最終更新日・本文要約）
- `main` ブランチに変更が push されると、`.github/workflows/deploy-pages.yml` によって自動的に次が行われます：
  1. GitHub API から最新の Issue 一覧を取得し、`web/_data/issues.json` に保存
  2. `web/` をソースとして Jekyll で静的サイトをビルド
  3. ビルド結果を GitHub Pages にデプロイ
- そのため、通常の開発フロー（Issue → ブランチ → PR → マージ）に従うだけで、ポータルサイトも自動で最新状態に保たれます。
