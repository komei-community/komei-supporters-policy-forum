# 運用ルール

## 1. 参加に関するルール

### 参加形態

ハイブリッド（誰でも参加し Issue/PR は作成できるが、マージは承認者のみ）

### 参加者の役割

コントリビューター（提案・コメント）

メンテナー（マージ権限、成果物公開管理）

## 2. 議論の進め方

### Issue の使い方

新規政策提案 → Issue テンプレートで作成

### ブランチの使い方

main = 公開版（採択済み）

issue/xx = Issue 作成で自動作成

## 3. プルリクのルール

### PR 作成時のテンプレート

背景 / 提案内容 / 期待効果

### レビュー観点

実現可能性

新規性・独自性

期待効果

### マージルール

マージには、運営管理者 2 名以上が必要

モデレーター 1 名以上の確認必須

「見送り」ラベルを付けてクローズする場合も履歴は残す

## 4. 成果物の管理

### フォルダ構成

sources/ … 新聞記事やマニフェストなど外部資料（Markdown 化）

policy-drafts/ … 提案ドラフト

policy/ … 提案中の政策ファイル置き場

adopted-policy/ … 採択済みの政策提案

research/ … データ・参考文献・統計など

web/ … GitHub Pages 公開用

### 採択後の扱い

adopted-policy/ に保存 → GitHub Pages に自動反映

定期的に「まとめレポート」を生成（Action で PDF/HTML 化）

公明党議員・関係者へのフィードバック資料に活用

## 5. 公明党との連携方法

定期的に「採択された提案まとめ」をメール/会合で共有

公明議員との意見交換会をオンライン開催（3 か月ごとなど）

公明新聞や広報誌に「オープン議論の成果」として取り上げてもらう働きかけ

## 6. モデレーション・秩序維持

### 行動規範（Code of Conduct）

批判は代替案とセットで

人格攻撃や差別を禁止

### 違反時の対応

コメント削除 → 警告 → アカウント制限の段階制

### ラベル管理

提案 / レビュー中 / 採択済 / 見送り / 要調整

## 7. 技術的運用

GitHub Pages → 成果物を誰でも閲覧可能に
